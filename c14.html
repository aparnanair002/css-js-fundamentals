DOM Manipulation — Quick Notes

1. What is the DOM?

The Document Object Model (DOM) is a programming interface that represents an HTML document as a tree of objects. Each HTML element, attribute, and piece of text becomes a node in this tree.

JavaScript can access and modify this tree to change the structure, content, and appearance of a web page dynamically.

2. DOM Tree Structure

  • The DOM is organized as a tree with a single root: document.
  • Each element is a node and may have child nodes.
  • Nodes have parent–child and sibling relationships.
  • Common node types:
    • Element nodes (Node.ELEMENT_NODE)
    • Text nodes (Node.TEXT_NODE)
    • Comment nodes (Node.COMMENT_NODE)

3. Accessing DOM Elements

  • document.getElementById() – selects an element by ID.
  • getElementsByTagName() – selects elements by tag name.
  • getElementsByClassName() – selects elements by class.
  • querySelector() – selects the first matching element.
  • querySelectorAll() – selects all matching elements.

4. Traversing the DOM

  • parentNode – access the parent element.
  • childNodes – access all child nodes.
  • children – access only element children.
  • firstChild / lastChild
  • previousSibling / nextSibling

5. Reading & Changing Content

  • textContent – reads or changes text inside an element.
  • innerHTML – reads or replaces HTML content.
  • nodeValue – accesses text node values.
  • Changing content updates the page immediately.

6. Creating & Removing Elements

  • document.createElement() – creates a new element.
  • document.createTextNode() – creates text nodes.
  • appendChild() – adds a node at the end.
  • insertBefore() – inserts a node at a specific position.
  • remove() – removes a node from the DOM.
  • replaceChild() – replaces one node with another.

7. Working with Attributes

  • getAttribute() – reads an attribute.
  • setAttribute() – sets or updates an attribute.
  • Custom attributes should use data-* format.
  • className is used to access the class attribute.

8. Styling with DOM

  • element.style modifies inline styles.
  • CSS property names use camelCase in JavaScript.
  • Example: style.backgroundColor
  • display: none hides elements without removing them.

9. Live vs Static Collections

  • getElementsBy* returns live collections.
  • querySelectorAll() returns static collections.
  • Live collections update automatically when DOM changes.
  • Use Array.from() to convert collections into arrays.

10. Performance Considerations

  • Frequent DOM reads and writes are expensive.
  • Avoid repeatedly forcing layout calculations.
  • Batch DOM updates when possible.
  • Use requestAnimationFrame() for animations.

11. Summary

DOM manipulation allows JavaScript to dynamically control web pages. Understanding the DOM tree, node traversal, and efficient updates is essential for building interactive and performant web applications.
