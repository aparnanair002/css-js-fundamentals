Node.js

Introduction

Node.js allows JavaScript to run outside the browser, making it useful for scripting, command‑line tools, and web servers.

Running JavaScript in Node

Save a file hello.js and run it with node hello.js. 


let message = "Hello world";
console.log(message);

Node’s REPL allows interactive JS execution.

Global Objects

  • console — Logging output.
  • process — Provides access to arguments and environment.

Modules

Node supports CommonJS and ES modules. Use require() or import to include functionality.


// ES module example
import { reverse } from "./reverse.mjs";
console.log(reverse("Node"));

NPM (Node Package Manager)

NPM installs packages from the online registry for reuse in your projects.


$ npm install ini
import { parse } from "ini";

Filesystem Module

The node:fs module provides async file operations.


import { readFile } from "node:fs";

readFile("file.txt", "utf8", (err, text) => {
  console.log(text);
});

HTTP Module

Node can run an HTTP server using the built‑in node:http module.


import { createServer } from "node:http";

createServer((req, res) => {
  res.writeHead(200, {"Content-Type": "text/html"});
  res.end("
Hello!
");
}).listen(8000);

Streams

  • Readable and writable streams handle large data.
  • Event‑based reading uses methods like on("data").

Building a File Server

A file server can map URLs to files and return their contents or directory listings.

Summary

  • Node extends JS beyond the browser.
  • Asynchronous I/O is a core design.
  • Modules and NPM make code reusable.
  • Useful built‑ins include filesystem, HTTP servers, and streams.
