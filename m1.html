Cascade
CASCADE IS SUCCESSION OF SOMETHINGS [ 0,0,0 ] Universal selector → * { } [ 0,0,1 ] Element/Tag selectors → p, h1, div [ 0,1,0 ] Class selectors → .intro, .btn [ 0,1,0 ] Attribute selectors → [type="text"] [ 0,1,0 ] Pseudo-classes → :hover, :nth-child() [ 1,0,0 ] ID selectors → #header [ 1,0,0+ ] Inline styles → style="color: red" [ ! ] !important → Overrides all normal rules
CSS Cascade & Specificity — Summary When an element matches multiple CSS rules, the browser collects all applicable rules and resolves conflicts using the cascade algorithm. If two rules set the same property, the more specific selector wins: Tags (low specificity) Classes (medium specificity) IDs (high specificity) Inline styles (higher) !important (highest) Non-conflicting styles are merged. In the example: css Copy Edit p { font-weight: bold; color: gray; } .introduction { color: violet; } → The paragraph gets font-weight from p and color from .introduction (class overrides tag). This process is like JavaScript object merging with spread syntax: js Copy Edit const appliedStyles = { ...tagStyles, ...classStyles }; Later (more specific) styles overwrite earlier ones. Deep specificity rules (like comparing #about-page button:last-of-type vs .form > button#submit) can be calculated, but modern development often makes exact specificity calculation less critical thanks to better tooling and CSS architecture patterns.

Box Model
The box model is a fundamental concept in CSS that describes how elements are structured and how their dimensions are calculated. The four main properties of the box model are:
  • Margin: The space between the box and other elements.
  • Border: The line surrounding the box.
  • Padding: The space between the content and the border.
  • Content: The actual content inside the box.
