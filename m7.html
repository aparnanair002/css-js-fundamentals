Grid Flow and Layout Modes
Changing grid-auto-flow from row to column doesn’t alter the grid orientation; it simply changes how new items are placed — multiple columns instead of multiple rows.
Grid Construction

grid-template-rows: 100px 200px 100px;
grid-template-columns: 100px 200px 100px;
You can also use other units like fr, auto, %, vw, vh.

If we include more than 3 children, the browser creates a 4th row and squeezes it in without overflow.

Use repeat() to simplify repetitive grid columns.

Alignment

Alignment properties for Grid containers:

  • justify-content: start | end | center | stretch | space-between | space-around | space-evenly
  • align-content: start | end | center | stretch | space-between | space-around | space-evenly
  • justify-items: start | end | center | stretch
  • align-items: start | end | center | stretch

align-items and align-self control cross-axis alignment.
align-content and justify-content handle extra space distribution.

grid-template-areas allows naming grid regions and assigning children via grid-area.

Minmax

The flexible unit must come last — minmax(1fr, 250px) is invalid because 1fr isn’t a valid minimum.

Autofill and Autofit

Using repeat() with auto-fill or auto-fit automatically creates as many columns (or rows) as fit the container.

Combine with minmax() for flexibility:

auto-fill keeps empty columns visible.
auto-fit collapses empty columns for tighter layouts.

Fluid Layout

Create fluid grids using min() and auto-fill for responsive columns.
