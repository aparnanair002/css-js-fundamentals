Grid Flow and Laout modes
When we change grid-auto-flow from row to column, we're not fundamentally changing the orientation of our grid; everything stays the same, except for the fact that our grid will have multiple columns instead of multiple rows.
Grid Construction
grid-template-row: 100px 200px 100px;
grid-template-column: 100px 200px 100px;
also other units are available like fr, auto, %, vw, vh
What happens if we include more than 3 children? the browser creates a 4th row and squeezes it in. It doesn't cause an overflow, it just means that there's less available space for the other elements.
to repeat a column multiple times, we can use the repeat function. In this case, we want to repeat a column that takes up 1 fraction of the available space 5 times.
Alignment
For Grid containers, we have the following alignment properties available to us: justify-content: start | end | center | stretch | space-between | space-around | space-evenly;
align-content: start | end | center | stretch | space-between | space-around | space-evenly;
justify-items: start | end | center | stretch;
align-items: start | end | center | stretch;
align-items is used on the parent to control the cross-axis position for all of the elements. But we also have align-self, which allows a specific child to overrule it align-content is used when there is extra space in the grid container. It controls how that extra space is distributed. justify-content does the same thing, but for the main axis. justify-items works like align-items, but for the main axis.
grid-template-areas: 'sidebar header' 'sidebar main-content'; this allows us to name different areas of our grid, and then assign children to those areas using the grid-area property.
Minmax
autofill and autofit When using the repeat function with the auto-fill or auto-fit keywords, the grid will automatically create as many columns (or rows) as will fit in the container, based on the size of the items.
Putting them together with minmax:
autofill and autofit difference: With auto-fill, the grid will create as many columns (or rows) as will fit in the container, even if some of those columns are empty. With auto-fit, the grid will collapse any empty columns, so that there are no gaps in the layout. The Fluid Layout
