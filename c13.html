JavaScript and the Browser — Quick Summary

1. The Web & JavaScript

JavaScript exists mainly because of web browsers. The web was designed as a universal, decentralized information space where anyone can share and link information.

2. Networks & the Internet

  • Computers communicate by sending bits over networks.
  • Protocols define how data is sent and interpreted.
  • TCP ensures reliable, ordered communication.
  • Clients connect to servers using ports (e.g., HTTP → port 80).

3. HTTP & URLs

HTTP is used to request and deliver web resources like pages and images. Each resource is identified using a URL:

  • Protocol: http / https
  • Server: domain name
  • Path: specific resource

4. HTML Basics

  • HTML structures web pages using tags.
  • Common tags: <h1>, <p>, <a>
  • Browsers are error-tolerant and fix missing tags.
  • Special characters use entities like &lt; and &amp;.

5. HTML & JavaScript

  • <script> allows JavaScript inside HTML.
  • Scripts can be inline or loaded from external files.
  • JavaScript can respond to user actions like clicks.

6. Browser Sandboxing

Browsers run JavaScript in a sandbox to protect users. Scripts cannot access files or system resources directly.

7. Browser Wars & Compatibility

  • Early browsers introduced incompatible features.
  • This caused major development issues.
  • Modern browsers follow standards more closely.
  • Today’s web is more stable and consistent.
