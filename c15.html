Handling Events

Event handling allows web pages to react to user interactions in real time.

  • Events are triggered by user actions or browser processes.
  • Event handlers are functions executed in response to these events.
  • Using handlers avoids constant polling for input.

Event Handlers and DOM Nodes

Handlers are attached to DOM nodes to react to specific events.

  • Use addEventListener to attach multiple handlers safely.
  • removeEventListener unregisters handlers.
  • Handlers can be attached to the window, document, or any element.
  • Handlers only respond to events on the node or its children (context matters).

Event Objects

Event handlers receive an object containing details about the event.

  • type: identifies the event type (e.g., "click").
  • target: the node where the event originated.
  • Properties vary by event type (mouse button, key pressed, coordinates).
  • Methods: stopPropagation() and preventDefault().

Propagation

Events bubble from the target element to its ancestors unless stopped.

  • Handlers on child nodes run before parent nodes.
  • Use stopPropagation() to prevent parent handlers from firing.
  • Useful for nested interactive elements to avoid conflicts.

Default Actions

Many events trigger default browser behavior.

  • Examples: clicking links, pressing keys, form submission.
  • Use preventDefault() to override default behavior.
  • Be cautious: breaking user expectations can frustrate users.

Key Events

Key events detect keyboard interactions.

  • keydown fires when a key is pressed; repeats if held down.
  • keyup fires when a key is released.
  • Check event.key for key identity and modifier keys (Shift, Ctrl, Alt, Meta).
  • Target depends on the focused element.

Pointer Events and Types

Pointer events include mouse and touch interactions.

  • Mouse: mousedown, mouseup, click, dblclick, mousemove.
  • Touch: touchstart, touchmove, touchend with multiple fingers.
  • Event objects provide coordinates, button codes, and touch points.
  • Useful for dragging, drawing, gestures, or custom UI controls.

Load Events

Load events signal when the page or resources have fully loaded.

  • window.onload or load event: page fully loaded.
  • Use for initialization after DOM and resources are ready.
  • beforeunload warns users before leaving the page.

Events and the Event Loop

Event handlers are executed asynchronously via the event loop.

  • Handlers wait for the main thread to be free.
  • Heavy or long-running handlers can block the page.
  • Web Workers can run code in parallel without blocking the UI.

Timers

Timers schedule code to run later or repeatedly.

  • setTimeout runs once after a delay.
  • setInterval runs repeatedly at a fixed interval.
  • requestAnimationFrame schedules animation updates efficiently.
  • Cancel with clearTimeout or clearInterval.

Debouncing

Debouncing prevents rapid events from executing handlers too frequently.

  • Useful for events like mousemove or scroll.
  • Handler execution is delayed until a pause occurs.
  • Use setTimeout and clear previous timeouts to implement.

Summary

Event handling in JavaScript is crucial for interactive web applications.

  • Attach handlers to DOM nodes using addEventListener.
  • Event objects provide detailed info about events and control methods.
  • Propagation allows parent nodes to react to child events; can be stopped.
  • Default actions can be prevented with preventDefault().
  • Key, mouse, touch, scroll, focus, and load events are commonly used.
  • Timers, debouncing, and Web Workers help manage performance and asynchronous tasks.

Exercises

  • Balloon: Control an emoji balloon using arrow keys. Inflate/deflate it and make it explode past a size limit.
  • Mouse Trail: Display elements following the mouse pointer with mousemove.
  • Tabs: Implement tabbed panels using data-tabname attributes and buttons to switch views.
