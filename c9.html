Regular Expressions – Summary

Regular expressions (regex) are patterns used for matching, searching, and manipulating text. They are concise but can be difficult to read.

1. Creating a Regex

let r1 = /abc/;
let r2 = new RegExp("abc");

Escape special characters using \.

2. Testing Matches

/abc/.test("abcde"); // true

3. Character Sets

  • [0-9] — digits
  • \d — digit
  • \w — word character
  • \s — whitespace
  • . — any char except newline

4. Unicode Properties

Available with the u flag:

  • \p{L} — letter
  • \p{N} — number

5. Repetition

  • + — one or more
  • * — zero or more
  • ? — optional
  • {n} — exact count
  • {n,m} — range

6. Grouping

(abc)   → capturing group
(?:abc) → non-capturing group

7. exec() & match()

/\d+/.exec("one 100"); // → ["100"]

8. Anchors & Boundaries

  • ^ — start of string
  • $ — end of string
  • \b — word boundary
  • (?= ) — look-ahead
  • (?! ) — negative look-ahead

9. Alternatives

/pig|cow|chicken/

10. Backtracking

Regex engines try multiple matching paths and backtrack when needed. Complex patterns may become slow.

11. replace()

"papa".replace("p","m"); // → "mapa"

Supports:

  • /g — global replacement
  • Groups like $1, $2
  • Replacement functions

12. Removing Comments Example

code.replace(/\/\/.*|\/\*[^]*?\*\//g, "");
